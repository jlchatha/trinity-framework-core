API Reference

The Trinity Framework 2.0 provides a comprehensive set of APIs for context management, workspace organization, agent collaboration, and more. This reference documentation covers all available APIs, their parameters, return values, and usage examples.

Note: The Trinity Framework 2.0 APIs are designed to be modular and extensible. You can use them individually or as integrated components of the full framework.

Context Management API

The Context Management API provides tools for tracking token usage, creating checkpoints, and managing context limitations.

Token Tracker Module

TokenTracker

Module: token-tracker/context-manager-v2/lib/token-counter.js

Purpose: Tracks token usage and manages context utilization

Methods

Method Parameters Returns Description
updateTokenCount(operationType) operationType: String - Type of operation Object - Updated context state Updates token count based on operation type
getContextUtilization() None Number - Percentage of context utilized Returns current context utilization percentage
getTokenCount() None Number - Current token count Returns the current token count
getMaxTokens() None Number - Maximum token capacity Returns the maximum token capacity
getRemainingOperations() None Object - Estimated remaining operations Returns estimated remaining operations of each size
resetTokenCount() None Object - Reset context state Resets token count to initial state

Usage Example

const TokenTracker = require('./lib/token-counter');

// Initialize token tracker
const tracker = new TokenTracker();

// Update token count for a user message
const state = tracker.updateTokenCount('USER_MEDIUM');
console.log(`Current utilization: ${tracker.getContextUtilization()}%`);

// Get remaining operations
const remaining = tracker.getRemainingOperations();
console.log(`Remaining small operations: ~${remaining.small}`);

Checkpoint Manager Module

CheckpointManager

Module: token-tracker/context-manager-v2/lib/checkpoint-manager.js

Purpose: Creates, stores, and restores checkpoints for context recovery

Methods

Method Parameters Returns Description
createCheckpoint(level, description) level: String - Checkpoint level
description: String - Description		 String - Checkpoint ID Creates a new checkpoint with current state
restoreCheckpoint(checkpointId) checkpointId: String - Checkpoint to restore Object - Restored state Restores state from the specified checkpoint
getCheckpoints() None Array - Available checkpoints Returns list of available checkpoints
getLatestCheckpoint() None Object - Latest checkpoint data Returns the most recent checkpoint
exportCheckpoint(checkpointId, outputPath) checkpointId: String - Checkpoint to export
outputPath: String - Export location		 Boolean - Success status Exports checkpoint to specified location
importCheckpoint(inputPath) inputPath: String - Path to checkpoint file String - Imported checkpoint ID Imports checkpoint from specified location

Usage Example

const CheckpointManager = require('./lib/checkpoint-manager');

// Initialize checkpoint manager
const checkpointManager = new CheckpointManager();

// Create a checkpoint
const checkpointId = checkpointManager.createCheckpoint('warning', 'Before implementing new feature');
console.log(`Created checkpoint: ${checkpointId}`);

// List available checkpoints
const checkpoints = checkpointManager.getCheckpoints();
console.log(`Available checkpoints: ${checkpoints.length}`);

Context Visualization Module

ContextVisualizer

Module: token-tracker/context-manager-v2/bin/context-visualization.js

Purpose: Generates text-based visualizations of context state

Methods

Method Parameters Returns Description
generateVisualization(state) state: Object - Current context state String - Visualization of context Generates progress bar visualization
generateDetailedVisualization(state) state: Object - Current context state String - Detailed visualization Generates detailed context visualization
generateThresholdIndicator(state) state: Object - Current context state String - Threshold indicators Generates threshold indicator visualization

Usage Example

const ContextVisualizer = require('./context-visualization');
const TokenTracker = require('../lib/token-counter');

// Initialize token tracker and visualizer
const tracker = new TokenTracker();
const visualizer = new ContextVisualizer();

// Update token count and visualize
const state = tracker.updateTokenCount('ASSISTANT_MEDIUM');
const visualization = visualizer.generateVisualization(state);
console.log(visualization);

Workspace Management API

The Workspace Management API provides tools for creating, organizing, and verifying agent workspaces.

Workspace Verifier Module

WorkspaceVerifier

Module: tools/workspace/workspace-verifier.js

Purpose: Verifies workspace structure and protocol file compliance

Methods

Method Parameters Returns Description
verifyWorkspace(workspacePath) workspacePath: String - Path to workspace Object - Verification results Performs comprehensive workspace verification
verifyProtocolFiles(workspacePath) workspacePath: String - Path to workspace Object - Protocol file verification results Verifies protocol files existence and content
verifyDirectoryStructure(workspacePath) workspacePath: String - Path to workspace Object - Directory structure verification results Verifies required directories exist
verifyNamingConventions(workspacePath) workspacePath: String - Path to workspace Object - Naming convention verification results Verifies files follow naming conventions

Usage Example

const WorkspaceVerifier = require('./tools/workspace/workspace-verifier');

// Initialize workspace verifier
const verifier = new WorkspaceVerifier();

// Verify workspace
const results = verifier.verifyWorkspace('/path/to/workspace');

// Check verification results
if (results.compliant) {
console.log('Workspace is Framework 2.0 compliant');
} else {
console.log('Compliance issues found:', results.issues);
}

Workspace Creator Module

WorkspaceCreator

Module: tools/workspace/workspace-creator.js

Purpose: Creates and initializes agent workspaces

Methods

Method Parameters Returns Description
createWorkspace(options) options: Object - Workspace options Object - Creation results Creates a new agent workspace
createProtocolFiles(workspacePath, agentRole) workspacePath: String - Path to workspace
agentRole: String - Agent role		 Object - Creation results Creates protocol files for the agent role
createDirectoryStructure(workspacePath, agentRole) workspacePath: String - Path to workspace
agentRole: String - Agent role		 Object - Creation results Creates directory structure for the agent role
installTools(workspacePath, agentRole) workspacePath: String - Path to workspace
agentRole: String - Agent role		 Object - Installation results Installs tools for the agent role

Usage Example

const WorkspaceCreator = require('./tools/workspace/workspace-creator');

// Initialize workspace creator
const creator = new WorkspaceCreator();

// Create workspace options
const options = {
workspacePath: '/path/to/new/workspace',
agentRole: 'HISTORIAN',
installTools: true,
createExamples: true
};

// Create workspace
const result = creator.createWorkspace(options);
console.log(`Workspace created: ${result.success}`);

Agent Collaboration API

The Agent Collaboration API provides tools for communication and coordination between specialized agents.

Agent Communication Module

AgentCommunicator

Module: tools/collaboration/agent-communicator.js

Purpose: Facilitates communication between agents through standardized interfaces

Methods

Method Parameters Returns Description
sendMessage(targetAgent, message) targetAgent: String - Target agent
message: Object - Message content		 Object - Delivery status Sends message to target agent
receiveMessages() None Array - Received messages Retrieves messages sent to this agent
broadcastMessage(message) message: Object - Message content Object - Broadcast status Sends message to all agents
shareFile(targetAgent, filePath) targetAgent: String - Target agent
filePath: String - File to share		 Object - Sharing status Shares file with target agent

Usage Example

const AgentCommunicator = require('./tools/collaboration/agent-communicator');

// Initialize agent communicator
const communicator = new AgentCommunicator('HISTORIAN');

// Send message to APEX
const message = {
type: 'documentation_update',
content: 'API documentation has been updated',
timestamp: new Date().toISOString()
};

const result = communicator.sendMessage('APEX', message);
console.log(`Message sent: ${result.success}`);

Recovery API

The Recovery API provides tools for context recovery and operational continuity after context resets.

Auto-Compact Detector Module

AutoCompactDetector

Module: token-tracker/context-manager-v2/bin/recovery/auto-compact-detector.js

Purpose: Detects auto-compact events and prepares for recovery

Methods

Method Parameters Returns Description
checkForAutoCompact() None Object - Detection results Checks if auto-compact has occurred
startMonitoring(options) options: Object - Monitoring options Object - Monitoring status Starts continuous monitoring for auto-compact
stopMonitoring() None Object - Stop status Stops continuous monitoring
prepareRecovery() None Object - Preparation results Prepares system for recovery after auto-compact

Usage Example

const AutoCompactDetector = require('./recovery/auto-compact-detector');

// Initialize auto-compact detector
const detector = new AutoCompactDetector();

// Check for auto-compact event
const result = detector.checkForAutoCompact();

// Handle detection result
if (result.detected) {
console.log('Auto-compact detected!');
detector.prepareRecovery();
} else {
console.log('No auto-compact detected');
}

Recovery Manager Module

RecoveryManager

Module: token-tracker/context-manager-v2/bin/recovery/recovery-manager.js

Purpose: Manages recovery procedures after context resets

Methods

Method Parameters Returns Description
initiateRecovery() None Object - Recovery results Initiates recovery after context reset
restoreFromCheckpoint(checkpointId) checkpointId: String - Checkpoint to restore Object - Restoration results Restores system state from checkpoint
updateProtocolFiles() None Object - Update results Updates protocol files with recovery information
generateRecoveryReport() None Object - Recovery report Generates detailed recovery report

Usage Example

const RecoveryManager = require('./recovery/recovery-manager');

// Initialize recovery manager
const recoveryManager = new RecoveryManager();

// Initiate recovery process
const result = recoveryManager.initiateRecovery();

// Check recovery status
if (result.success) {
console.log(`Recovery successful, restored from ${result.checkpointId}`);
} else {
console.log('Recovery failed:', result.error);
}
