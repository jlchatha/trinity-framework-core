Glossary

This glossary provides definitions for terminology used throughout the Trinity Framework 2.0 Core documentation. Having a standardized vocabulary ensures clear communication across all components of the framework.

Note: Consistent terminology is essential for context resilience. Using standard terms reduces ambiguity and improves operational clarity across context resets.

Framework Terminology

Framework 2.0

The second generation of the Trinity Framework, featuring standardized workspace organization, protocol-based recovery, and context management. Framework 2.0 introduces formal agent specialization and collaboration patterns.

Related Terms: Trinity Framework, Protocol Files, Context Resilience

Context Resilience

The ability to maintain operational continuity across context resets. Achieved through standardized protocols, checkpointing, and knowledge preservation mechanisms that enable rapid recovery and continuation of work.

Related Terms: Context Management, Checkpointing, Auto-Compact Recovery

Workspace Standardization

The practice of maintaining consistent directory structures, file naming conventions, and organizational patterns across all agent workspaces to ensure predictability and operational efficiency.

Related Terms: Directory Structure, Protocol Files, Workspace Organization

Protocol-Based Recovery

A formalized approach to recovering operational awareness after context resets, using standardized procedures defined in protocol files to restore functionality efficiently.

Related Terms: Auto-Compact Recovery, Checkpointing, Context Management

Agent Roles

🔷

APEX

The strategic coordination agent in the Trinity Model, responsible for high-level architecture, project direction, and system integration. APEX takes a "big picture" view, ensuring all components work together cohesively.

Key Characteristics: Strategic thinking, architectural expertise, system-level vision

Related Terms: Strategic Coordination, System Architecture, Trinity Model

🔶

ANALYST

The analytical implementation agent in the Trinity Model, responsible for detailed implementation, data analysis, and tactical execution. ANALYST translates APEX's strategic vision into concrete implementations.

Key Characteristics: Technical expertise, analytical thinking, implementation focus

Related Terms: Technical Implementation, Data Analysis, Code Optimization

📘

HISTORIAN

The documentation and knowledge preservation agent in the Trinity Model, responsible for maintaining comprehensive documentation, knowledge organization, and context continuity across the system.

Key Characteristics: Documentation expertise, knowledge organization, communication clarity

Related Terms: Documentation, Knowledge Preservation, AAR System

Protocol Files

AGENT.md

The protocol file that defines an agent's identity, role, responsibilities, and recovery procedures. It serves as the primary reference point for an agent to understand its purpose and operational parameters.

Key Sections: Identity & Mission, Core Principles, Auto-Compact Recovery Protocol, Key Paths and Locations

Related Terms: Agent Identity, Recovery Protocol, Framework Guidelines

STATUS.md

The protocol file that tracks current operational status, project progress, and priorities. It serves as a comprehensive status dashboard that is regularly updated to reflect the latest state of operations.

Key Sections: Framework Status, Project Status, Next Priorities, Recent Accomplishments, Context Health

Related Terms: Operational Status, Project Progress, Context Health

AUTO-COMPACT.md

The protocol file dedicated to context management, containing information about current context utilization, checkpoint data, and recovery instructions. It plays a critical role in maintaining operational continuity across context resets.

Key Sections: Context Management Metrics, Checkpoint Data, Checkpoint Restoration Instructions, Current Implementation Context

Related Terms: Context Management, Checkpointing, Auto-Compact Recovery

Context Management

Context

The available memory or token capacity within a language model's operational window. Context represents the information an agent can hold at one time before requiring a reset (auto-compact).

Related Terms: Token Utilization, Context Window, Context Reset

Token

The basic unit of context utilization, representing a piece of text processed by the language model. Tokens are used to measure context consumption and estimate remaining capacity.

Related Terms: Token Counting, Context Utilization, Token Estimation

Checkpoint

A serialized snapshot of operational state created at specific thresholds or on demand. Checkpoints capture critical information that enables recovery after context resets.

Key Types: NOTICE (65%), WARNING (75%), ALERT (85%), MANUAL

Related Terms: Checkpoint Manager, State Serialization, Recovery

Auto-Compact

The process of resetting a language model's context when it approaches capacity limits. Auto-compact clears the model's memory, necessitating recovery procedures to restore operational awareness.

Related Terms: Context Reset, Auto-Compact Detection, Recovery Protocol

Threshold

Predefined context utilization levels that trigger specific actions, such as notifications, warnings, checkpoint creation, and recovery preparation.

Standard Thresholds:

  • NOTICE: 65% - Initial awareness of context growth
  • WARNING: 75% - Create checkpoint and prepare for potential compact
  • ALERT: 85% - Final checkpoint creation and imminent compact warning

Related Terms: Context Monitoring, Checkpoint Creation, Visualization

Workspace Organization

Directory Structure

The standardized organization of directories within an agent workspace, including required directories like tools/, AAR/, reports/, and checkpoints/, as well as role-specific directories.

Related Terms: Workspace Standardization, Directory Organization, File Structure

AAR (After Action Review)

A structured documentation format for preserving knowledge across context boundaries. AARs capture session learnings, incident analyses, and procedural improvements for future reference.

Types:

  • Session AAR: 20-50 line document covering one work session
  • Incident AAR: 100+ line comprehensive analysis of significant incidents

Related Terms: Knowledge Preservation, Documentation, Session Review

Tools

Specialized utilities and scripts that support agent operations, organized in the tools/ directory. Tools include context management, workspace verification, documentation generation, and role-specific utilities.

Key Categories: Context Management, Workspace Management, Documentation, Recovery

Related Terms: Utilities, Scripts, Automation

Project Management

UNIT

The smallest atomic operational cycle in the Trinity Framework's project management terminology, equivalent to a daily work cycle. A UNIT represents a single, cohesive work period.

Related Terms: Operational Cycle, MILESTONE, Work Period

MILESTONE

A collection of UNITs forming an achievement checkpoint, equivalent to a weekly work cycle. A MILESTONE represents a significant accomplishment within a project phase.

Composition: 7 UNITs

Related Terms: Achievement Checkpoint, UNIT, PHASE

ITERATION

A process improvement cycle focusing on refinement and optimization, equivalent to a fortnightly work cycle. An ITERATION represents a complete cycle of implementation and improvement.

Composition: 14 UNITs or 2 MILESTONEs

Related Terms: Process Improvement, Refinement Cycle, MILESTONE

PHASE

A major project segment with a specific focus or objective, equivalent to a monthly work cycle. A PHASE represents a substantial component of a complete project.

Composition: 4 MILESTONEs or 2 ITERATIONs or 28 UNITs

Related Terms: Project Segment, MILESTONE, PROJECT

PROJECT

A complete initiative with defined objectives, scope, and deliverables, equivalent to a quarterly work cycle. A PROJECT represents a full implementation of a feature, system, or capability.

Composition: 3 PHASEs or 12 MILESTONEs or 84 UNITs

Related Terms: Initiative, Complete Implementation, PHASE

Technical Concepts

Trinity Model

A three-agent collaborative model with specialized roles (APEX, ANALYST, HISTORIAN) working together as a cohesive system. The Trinity Model distributes cognitive workload across specialized agents while maintaining operational alignment.

Related Terms: Multi-Agent Collaboration, Agent Specialization, Role Division

Operation Type

A categorization of actions based on their context consumption, used for token estimation and context tracking. Operation types include user messages, assistant responses, and tool operations of varying sizes.

Examples: USER_MEDIUM, ASSISTANT_LARGE, TOOL_READ_SMALL

Related Terms: Token Estimation, Context Tracking, Context Management

Synchronization

The process of aligning information across agent workspaces to ensure consistency and operational coordination. Synchronization includes status alignment, context metric sharing, and checkpoint coordination.

Related Terms: Workspace Synchronizer, Status Alignment, Coordination

Visualization

Text-based representation of context utilization and system state, providing clear feedback on current status and remaining capacity. Visualizations include progress bars, thresholds, and remaining operation estimates.

Example: [Context: 45% ▓▓▓▓▓▓▓▓▓░░░░░░░░░░░] Remaining: ~73 small operations

Related Terms: Context Visualization, Progress Bar, Threshold Indicator

Systems and Components

Context Management System

A comprehensive system for tracking token usage, creating checkpoints, and managing context limitations. The Context Management System includes token tracking, checkpoint management, visualization, and recovery components.

Key Components: TokenTracker, CheckpointManager, ContextVisualizer, RecoveryManager

Related Terms: Token Tracking, Checkpointing, Context Visualization

Token Tracker

A component of the Context Management System that estimates token usage for different operations and maintains a running count of context utilization. The Token Tracker provides information about current utilization and remaining capacity.

Related Terms: Context Management, Token Counting, Operation Types

Checkpoint Manager

A component of the Context Management System that creates, stores, and restores checkpoints for context recovery. The Checkpoint Manager ensures that critical state information is preserved across context resets.

Related Terms: Checkpointing, State Serialization, Recovery

Auto-Compact Detector

A component of the Context Management System that identifies when a context reset has occurred by monitoring context utilization patterns and timing. The Auto-Compact Detector triggers recovery procedures when a reset is detected.

Detection Methods: Utilization Drop, Time Heuristics

Related Terms: Context Reset, Recovery Trigger, Monitoring

Recovery Manager

A component of the Context Management System that manages recovery procedures after context resets, including checkpoint restoration, protocol file updates, and operational continuity maintenance.

Related Terms: Context Recovery, Checkpoint Restoration, Protocol Updates
