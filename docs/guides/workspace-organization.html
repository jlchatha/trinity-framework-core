Workspace Organization Guide

Proper workspace organization is crucial for context resilience and effective agent operations. This guide provides detailed instructions for setting up and maintaining a standardized Trinity Framework 2.0 workspace, ensuring that all agents can operate efficiently and recover quickly from context resets.

Note: Consistent workspace organization is not just about file management—it's a foundational requirement for context resilience and operational continuity.

Workspace Directory Structure

The Trinity Framework 2.0 requires a standardized directory structure for all agent workspaces. This structure ensures consistent organization across agents and facilitates smooth collaboration and context recovery.

/path/to/agent/workspace/
├── AGENT.md # Agent identity and recovery protocol
├── STATUS.md # Current operational status
├── AUTO-COMPACT.md # Context management protocol
├── tools/ # Utilities and scripts
│ ├── setup/ # Setup and configuration tools
│ ├── token-tracker/ # Context management tools
│ └── [role-specific]/ # Role-specific tools
├── AAR/ # After Action Review documents
│ ├── sessions/ # Session-based knowledge preservation
│ └── incidents/ # Incident analysis and resolution
├── reports/ # Analysis and status reports
├── checkpoints/ # Context checkpoints
└── [role-specific]/ # Role-specific directories

Setting Up a New Workspace

Follow these steps to set up a new Trinity Framework 2.0 workspace:

1. Create Base Directory Structure

$ mkdir -p /path/to/agent/workspace/{tools/{setup,token-tracker},AAR/{sessions,incidents},reports,checkpoints}
$ cd /path/to/agent/workspace
$ ls -la
total 8
drwxr-xr-x 6 user user 4096 Jun 13 09:15 .
drwxr-xr-x 4 user user 4096 Jun 13 09:15 ..
drwxr-xr-x 3 user user 60 Jun 13 09:15 AAR
drwxr-xr-x 2 user user 40 Jun 13 09:15 checkpoints
drwxr-xr-x 2 user user 40 Jun 13 09:15 reports
drwxr-xr-x 3 user user 60 Jun 13 09:15 tools

2. Create Protocol Files

Create the three required protocol files in the workspace root:

AGENT.md

Create this file with agent identity and recovery protocol:

Use the template from the Trinity Framework repository:

$ curl -s https://raw.githubusercontent.com/jlchatha/trinity-framework-core/main/templates/AGENT.md.template > AGENT.md
$ nano AGENT.md # Edit to customize for your agent

STATUS.md

Create this file with current operational status:

Use the template from the Trinity Framework repository:

$ curl -s https://raw.githubusercontent.com/jlchatha/trinity-framework-core/main/templates/STATUS.md.template > STATUS.md
$ nano STATUS.md # Edit to reflect current status

AUTO-COMPACT.md

Create this file with context management protocol:

Use the template from the Trinity Framework repository:

$ curl -s https://raw.githubusercontent.com/jlchatha/trinity-framework-core/main/templates/AUTO-COMPACT.md.template > AUTO-COMPACT.md
$ nano AUTO-COMPACT.md # Edit context management settings

3. Install Context Management Tools

Set up the token tracking and context management system:

$ git clone https://github.com/jlchatha/trinity-framework-core.git
$ cp -r trinity-framework-core/tools/token-tracker/* tools/token-tracker/
$ cd tools/token-tracker/context-manager-v2/bin
$ ./setup/install-dependencies.sh
Installing dependencies for context management system...
✓ Context management system installed successfully

4. Create Role-Specific Directories

Based on your agent's role, create the appropriate role-specific directories:

Agent Role Command Directories Created
APEX mkdir -p architecture planning integration Strategic planning and architectural documentation
ANALYST mkdir -p analysis data implementations Data analysis and implementation work
HISTORIAN mkdir -p documentation templates archives Documentation management and knowledge preservation

Maintaining Workspace Organization

Regular Maintenance Tasks

To maintain a clean and efficient workspace, perform these tasks regularly:

  • Daily: Update STATUS.md with current progress and priorities
  • After Each Session: Create AAR/sessions/ entry to document session outcomes
  • Before Context Reset: Create checkpoint with ./tools/token-tracker/context-manager-v2/bin/run-context-check.sh checkpoint
  • Weekly: Verify workspace structure with ./tools/verify-workspace.sh
  • Monthly: Clean up unnecessary files and archives

File Naming and Organization Conventions

Consistent file naming and organization is essential for context resilience:

Item Type Naming Convention Examples
Protocol Files UPPERCASE.md AGENT.md, STATUS.md, AUTO-COMPACT.md
Directories lowercase-with-hyphens/ token-tracker/, system-analysis/
Tool Scripts verb-noun.extension verify-workspace.sh, update-docs.js
Session AARs YYYY-MM-DD-description.md 2025-06-13-api-documentation.md
Incident AARs DESCRIPTIVE-NAME-RCA.md API-AUTHENTICATION-FAILURE-RCA.md
Reports report-type-YYYYMMDD.md status-report-20250613.md
Checkpoints ckpt_level_YYYYMMDD_n ckpt_warning_20250613_1

Workspace Anti-Patterns

Important: Avoid these common workspace organization anti-patterns:

  • Protocol File Fragmentation: Splitting protocol files or placing them in subdirectories
  • Inconsistent Naming: Using different naming conventions across files and directories
  • Tool Proliferation: Creating too many one-off tools without proper organization
  • AAR Neglect: Failing to create AAR entries for significant sessions
  • Checkpoint Scarcity: Not creating checkpoints before approaching context limits
  • Directory Sprawl: Creating too many top-level directories without clear purpose
  • Role Confusion: Maintaining directories that belong to another agent's role

Multiple Workspace Management

In scenarios where you manage multiple workspaces for different projects:

  1. Use Absolute Paths: Always use absolute paths in protocol files and scripts
  2. Implement Workspace Switcher: Create a script to quickly switch between workspaces
  3. Maintain Consistency: Use identical organization patterns across all workspaces
  4. Context Isolation: Clearly separate context tracking between workspaces
  5. Cross-References: Document cross-workspace references in STATUS.md
$ cat > ~/workspace-switcher.sh << 'EOF'
#!/bin/bash
case $1 in
apex) cd ~/trinity-apex-workspace ;;
analyst) cd ~/trinity-analyst-workspace ;;
historian) cd ~/trinity-historian-workspace ;;
*) echo "Usage: workspace-switcher.sh {apex|analyst|historian}" ;;
esac
EOF
$ chmod +x ~/workspace-switcher.sh

Best Practices

  • Protocol File Primacy: Always ensure protocol files are up-to-date and complete
  • Single Responsibility: Each directory should have a clear, single purpose
  • Depth over Breadth: Prefer deeper directory structures over many top-level directories
  • Regular Verification: Periodically run workspace verification tools
  • Consistent Navigation: Use consistent paths when referencing files across the workspace
  • Documentation First: Document workspace changes before implementing them
  • Role Appropriateness: Organize directories according to your agent's specific role
