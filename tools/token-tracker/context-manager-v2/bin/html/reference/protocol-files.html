Protocol Files Reference

Protocol files form the foundation of the Trinity Framework 2.0, providing a standardized structure for agent identity, status tracking, and context management. These files enable context resilience and operational continuity across context resets.

Note: All Trinity Framework 2.0 compliant workspaces must include these protocol files in the workspace root directory.

Core Protocol Files

AGENT.md

Purpose: Defines agent identity, mission, and recovery procedures

The AGENT.md file serves as the primary identity document for any agent in the Trinity Framework. It contains the agent's role, responsibilities, and auto-compact recovery protocol. When a context reset occurs, this is the first file the agent should read to restore its operational awareness.

Key Sections

  • Role and Identity: Defines the agent's name, role, and primary purpose
  • Core Principles: Lists the framework principles the agent adheres to
  • Auto-Compact Recovery Protocol: Step-by-step recovery procedure after context resets
  • Mission and Responsibilities: Detailed description of the agent's specific duties
  • Workflow: Standard operational procedures for daily tasks

STATUS.md

Purpose: Tracks current operational status, priorities, and progress

The STATUS.md file maintains a record of the agent's current operational state, ongoing projects, recent accomplishments, and immediate priorities. It provides the temporal context necessary for the agent to resume work after interruptions or context resets.

Key Sections

  • Current Status: Overall operational state of the agent
  • Active Projects: List of in-progress work with completion percentages
  • Recent Accomplishments: Completed tasks and milestones
  • Next Priorities: Ordered list of pending tasks by priority
  • Context Health: Metrics on context window utilization

AUTO-COMPACT.md

Purpose: Manages context utilization and provides recovery mechanism

The AUTO-COMPACT.md file contains the context management protocol for monitoring token usage, creating checkpoints, and enabling smooth recovery after context resets. This file is critical for maintaining operational continuity in long-running agent sessions.

Key Sections

  • Context Management Metrics: Current utilization and capacity estimates
  • Checkpoint Data: Serialized state for recovery purposes
  • Recovery Instructions: Procedures for restoring state after reset
  • Implementation Context: Current work context for continuity

Protocol File Implementation

To implement the protocol files in your own Trinity Framework 2.0 project:

  1. Create the three protocol files (AGENT.md, STATUS.md, AUTO-COMPACT.md) in your workspace root
  2. Customize the AGENT.md file with your agent's specific role and responsibilities
  3. Initialize the STATUS.md file with your current project status
  4. Set up the AUTO-COMPACT.md file with the appropriate context management configuration
  5. Ensure all agents follow the recovery protocol after context resets

Warning: Modifying the core structure of protocol files may compromise the framework's context resilience capabilities. Always maintain the required sections and update frequencies.
